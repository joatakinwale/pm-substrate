# D6 evidence artifacts

Boundary-conformance artifacts in this directory are generated by
`pnpm pm:boundary`. They contain no raw command output: stdout and stderr are
represented by byte counts and SHA-256 hashes, so the receipt can be reviewed
without copying possible secrets into the repository.

A red artifact is durable falsification evidence, not a valid D6 credential.
`pnpm pm:objective -- record` recomputes the artifact content hash and refuses
an artifact unless it is green and its app ID, app revision/source-tree hash,
and substrate revision exactly match the measurement.

Gitless app snapshots must declare every source root consumed by their checks
with repeated `--fingerprint-path` flags. The sorted scope is encoded directly
in the resulting `appRevision`; generated state, dependency directories, and
common build caches are excluded.

Artifacts cited by an objective measurement must be stored inside this
pm-substrate checkout (normally here). The memo refuses external paths and
symlinks that escape the checkout, so an untrusted event cannot turn evidence
verification into an arbitrary local-file read.
