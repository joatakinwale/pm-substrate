STEP 2 — Project Selection (approval due Week 4)
Chosen topic (winner from Step 1) PM Substrate for Reliable AI Agent Project Workflows — applying project management methods to the planning, implementation, and monitoring of the pm-substrate repository
Project type Case study of a PM methodology
Organization / context (real or hypothetical) Real, individually-run open-source software project: the pm-substrate GitHub repo (typed events, graph-based state, read-set validation, observation contracts, eval scenarios, Local Agents Lab Enviroment as TestBed then a Financial Hedge Fund for research and trading - which requests high in/out flow of data constantly,  finance-adapter example). Peer input welcomed on scope, PM metrics, and non-technical presentation.
Why it matters to project management AI agents often act on stale, incomplete, or non-authoritative information. This project demonstrates how core PM disciplines — budgeting, scheduling, resource allocation, execution tracking, monitoring & control, and transition to operations — keep a fast-changing technical system correct, traceable, and risk-controlled.
One-sentence topic statement A case study applying advanced project management methods — budgeting, scheduling, resource allocation, execution monitoring, and operational transition — to the delivery of pm-substrate, an open-source system that gives AI agents a validated, current-state view of a project before they act.
Rubric target (20 pts): “highly relevant, clearly defined, demonstrates strategic depth.”
STEP 3 — Objectives (what your project must demonstrate)
Overall project objective (SMART) By the Week 13 poster deadline, deliver an evidence-based case study of the pm-substrate project that documents: (1) a complete project plan — time/tooling budget, milestone schedule, and resource plan; (2) execution tracked through commits, tests, and changelog; (3) monitoring via build verification and eval-scenario metrics; and (4) a transition-to-operations package (versioned release + runbook) — with topic approval secured by Week 4.

(MLO 1, 2; CLO A, E) Poster narrates the full lifecycle of pm-substrate — planning (architecture, milestones, risk register) through execution (tracked code changes, tests, changelog) — under an explicit management approach: iterative delivery with stage-gate milestone reviews.
Objective 2 Develop a comprehensive project plan that includes budgeting, scheduling, and resource allocation. (MLO 2, 2;CLO D, E) Plan built in this workbook: time/tooling budget for implementation and testing phases; milestone schedule (research → architecture → adapter → validation rules → eval scenarios → reporting); resource plan across five roles — engineering, PM, testing, technical writing, domain review.
Objective 3 Analyze a complex project and develop documentation of the process  for transitioning the project to operational status. (MLO1, 2; CLO D, E) Analysis of monitoring data (test results, build verification, eval metrics) plus a documented transition to operational status: versioned release, runbook/usage documentation, and acceptance criteria for ongoing use.
