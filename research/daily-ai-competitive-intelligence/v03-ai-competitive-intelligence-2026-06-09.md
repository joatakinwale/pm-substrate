## 1. Delta From v02

v03 does not overturn v02's conclusion. It sharpens it.

The newest official releases show the major vendors moving three layers closer to pm-substrate's territory:

1. **Third-party agent validation is becoming platform behavior.** GitHub made security validation for third-party coding agents generally available on 2026-06-09. Code generated by agents such as Claude and OpenAI Codex is now automatically checked with CodeQL, dependency advisories, and secret scanning, and the agent may attempt fixes before finalizing the pull request. This is not pm-substrate-style state review, but it is a direct platform-level admission that external agents need pre-merge validation evidence.
2. **Model/provider policy is now action context.** GitHub and Anthropic both launched Claude Fable 5 on 2026-06-09. GitHub requires Copilot Business and Enterprise admins to enable a policy that is off by default because the model requires up to 30 days of data retention for Anthropic safety classifiers. This turns model selection, provider terms, data retention, and admin policy into first-class review facts.
3. **Agent-capable models are spreading into more client surfaces.** Google updated its Gemini-for-Apple-developers announcement on 2026-06-09, bringing Gemini models through Apple's Foundation Models framework and into Xcode. This expands the number of clients where agentic code or app actions can originate, but does not by itself add currentness, authority, or workflow validation.

The main competitive correction is that "agent governance" is now too broad a phrase. The freshest evidence separates into at least five different evidence lanes:

- security validation evidence, such as CodeQL, dependency, and secret scans;
- model/provider policy evidence, such as admin opt-in, data-retention terms, and provider availability;
- client-surface evidence, such as Copilot app, Xcode, VS Code, Claude Code, Codex, and Gemini;
- runtime/session evidence, such as AgentCore traces, interactive shells, file systems, and eval loops;
- operational-state evidence, which remains the pm-substrate differentiator: original observation, current view, read-set validation, authority, workflow validity, and replayable action review.

No fresh official source reviewed on 2026-06-09 exposed a portable equivalent to `CurrentStateView + original ObservationContract + ReadSetValidation + ActionProposalReview + StateReviewArtifact` with policy outcome and replay hash.

## 3. Company-by-Company Delta

### GitHub and Microsoft

Fresh delta: GitHub's 2026-06-09 security validation for third-party coding agents is the most important competitive move in this run. It makes the platform, not only the agent, responsible for checking generated code before pull request finalization. The checks named in the official note are CodeQL analysis, GitHub Advisory Database dependency checks, and secret scanning. These validations are on by default and follow repository Copilot settings.

pm-substrate interpretation: this is external validation evidence. It proves that a platform can intercept agent output and attach automated review results before merge. It does not prove the agent's source observations were current, authoritative, tenant-correct, or workflow-valid when it wrote the code.

Threat level: High, increased confidence.

Implementation implication: add a `third_party_agent_validation` fixture where external security evidence is admitted into a state-review artifact as evidence, while still requiring substrate checks for source authority, stale reads, subject binding, and workflow position.

### Anthropic

Fresh delta: Anthropic launched Claude Fable 5 and Mythos 5 on 2026-06-09, with Claude Fable 5 generally available and positioned for hard knowledge-work and coding tasks. Anthropic's model docs say Fable 5 is available across Claude API, AWS Bedrock, Vertex AI, and Microsoft Foundry beginning the same day.

pm-substrate interpretation: the competitive issue is not just model capability. Fable 5's GitHub Copilot integration requires an explicit enterprise/admin policy because of data retention for safety classifiers. That makes model-provider policy, data-retention constraints, and deployment channel part of the evidence needed before an agent uses a model on sensitive work.

Threat level: Medium-high for model/workflow capability; High as a provider-policy evidence input.

Implementation implication: add model-provider policy fields to review artifacts or external evidence admission: `modelProvider`, `modelId`, `providerSurface`, `dataRetentionPolicyRef`, `adminPolicyRef`, `zdrStatus`, and `policyAcknowledgedAt`.

### OpenAI

Fresh delta: OpenAI's 2026-06-09 Nextdoor Codex customer story is an enterprise adoption proof point, not a new mechanism release. The stronger carry-forward OpenAI evidence remains the 2026-06-05 Enterprise/Edu release note for plugin sharing and the 2026-06-02 Codex/Sites updates.

pm-substrate interpretation: OpenAI is still a high distribution threat because Codex can package plugins, Sites, workflow bundles, and internal apps inside enterprise workspaces. The fresh June 9 source does not show portable pre-action state-review artifacts.

Threat level: High distribution/control-plane risk; no change to operational-state equivalence.

Implementation implication: keep Codex/plugin/provider/session metadata on artifacts and add plugin-policy drift fixtures before treating plugin distribution as safe.

### Google

Fresh delta: Google's Gemini-for-Apple-developers announcement was published 2026-06-08 and updated 2026-06-09. It puts Gemini models behind Apple's Foundation Models framework and integrates Gemini into Xcode. Google DeepMind's model-card page also shows a 2026-06-09 update for Gemini 3.5 Audio.

pm-substrate interpretation: Google is expanding client-surface and model-provider reach. The operational-state gap remains unchanged: a shared API surface and model card do not decide whether a proposed action is valid against current business state.

Threat level: Medium-high.

Implementation implication: add `clientSurface` and `providerLifecycleRef` to external evidence fixtures so the same action can be reviewed differently when it originates from Xcode, Copilot, Codex, Claude Code, or a hosted workflow runtime.

### AWS

Fresh delta: no new official 2026-06-09 AgentCore launch note was found, but current AgentCore docs remain strategically important. The release-note page now emphasizes production-trace optimization, batch evaluation, user simulation, file-system mounts, GovCloud availability, payments, custom headers, and the existing policy/memory/gateway/observability stack. The interactive-shell docs note that agents created after 2026-06-05 support terminals automatically and require `bedrock-agentcore:InvokeAgentRuntimeCommandShell`.

pm-substrate interpretation: AWS is still the strongest infrastructure/runtime comparator. AgentCore has the operational substrate for execution, traces, evals, identity, gateway, and policy. The open competitive question is whether it validates original observations and read sets against authoritative current state before business mutations.

Threat level: High, unchanged.

Implementation implication: treat AgentCore as a runtime/eval comparator and add fixtures that compare runtime trace/eval evidence against pm-substrate's action-review artifact.

## 7. pm-substrate Implications

1. **External validation is now a product category.** GitHub's third-party-agent validation gives pm-substrate a concrete comparator for validator evidence. The substrate should ingest validation outcomes but still decide action validity from current-state and authority checks.
2. **Provider policy belongs in the artifact.** Model capability, data-retention behavior, and admin enablement can change whether an agent may process a source or produce an action. These fields should not live only in prose.
3. **Client surfaces must be recorded.** A proposal from Codex, Copilot app, Claude Code, Xcode/Gemini, or AgentCore may carry different permissions, retention, session, and audit semantics.
4. **Runtime traces and evals are support evidence.** AgentCore-style production traces, batch evals, and simulations are strong evidence lanes, but they do not replace the original-observation/current-state boundary.
5. **Marketing "governance" should be split into precise claims.** Governance can mean admin policy, data retention, security validation, approvals, audit, runtime identity, or operational-state validity. pm-substrate should keep those separate in artifacts and ledgers.
